  - No formal appeal mechanism

================================================================================
ARCHITECTURE ASSESSMENT
================================================================================

STRENGTHS:
  ✓ Server-side enforcement (client checks are advisory)
  ✓ Fail-open safeguards (never silently approve dangerous content)
  ✓ Sacred/monitored separation (E2E for adults)
  ✓ Multi-layer defense (banned terms → Perspective → Vision → policy)
  ✓ Crisis never blocked (always delivers + resources)
  ✓ No hardcoded backdoors

WEAKNESSES:
  ✗ 4+ moderation entry points for posts (complexity)
  ✗ Privacy disclosure gaps (monitored DMs, comment rejection)
  ✗ No unified appeal or moderation history UI
  ✗ Async comment moderation (silent deletion)
  ✗ Shadow-ban capability (deceptive)
  ✗ Crisis detection not audited (no feedback loop)

OVERALL VERDICT: Production-ready with reservations. Fix critical issues before
scaling to 100K+ users (especially if collecting minor data).

COMPLIANCE CHECKLIST
================================================================================

Before shipping social features with minors:

COPPA (Children's Online Privacy Protection Act):
  [ ] Monitored DM disclosure in composer (CRITICAL FIX)
  [ ] Parental consent if applicable
  [ ] No sale of minor data to third parties
  [ ] Privacy policy link in app

GDPR (EU):
  [ ] User can request access to moderation decisions
  [ ] User can submit appeal
  [ ] Data retention policy documented
  [ ] Third-party processors disclosed

FTC (Unfair/Deceptive Practices Act):
  [ ] No shadow-banning without transparency (CRITICAL FIX)
  [ ] AI involvement disclosed
  [ ] Appeal mechanism exists
  [ ] Moderation decision explains rationale

CCPA (California Consumer Privacy Act):
  [ ] User can access/delete moderation records
  [ ] Third-party AI vendors listed
  [ ] No sale of moderation data
