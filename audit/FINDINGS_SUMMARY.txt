KEY FINDINGS

CRITICAL ISSUES (Act within 2 sprints)
--------------------------------------
1. [HIGH] Message Array Unbounded Growth (F-perf-001)
   - Location: BereanChatView.swift:115-116, 328, 463
   - Problem: messages[] array grows unbounded; no eviction for 50+ message convos
   - Impact: Memory leak in multi-hour conversations (150+ MB possible)
   - Risk: Graceful degradation expected (iOS memory warning cleanup), but not guaranteed
   - Fix: Implement message windowing; keep only last 30 in memory; lazy-load old messages from Firestore

2. [MEDIUM] BiblicalAlignmentService Blocking UI (F-perf-002)
   - Location: BereanChatView.swift:477-495
   - Problem: Post-stream alignment check blocks MainActor for 300-800ms
   - Impact: Perceptible "app froze" stall after response arrived
   - Fix: Move alignment check to background Task; update UI when ready

3. [MEDIUM] Preflight Latency Misreported (F-perf-004)
   - Location: ClaudeService.swift:502-572
   - Problem: First-token latency doesn't include 100-500ms preflight time
   - Impact: Product team sees 200ms when reality is 600-1200ms; SLOs silently missed
   - Evidence: startedAt=Date() is AFTER preflight completes
   - Fix: Move startedAt to start of sendBereanChatMessage; instrument preflight separately

MEDIUM ISSUES (Act within 4 sprints)
------------------------------------
4. [MEDIUM] ChatMemoryService Listener Never Cancelled (F-perf-003)
   - Location: BereanMemoryService.swift:41-58
   - Problem: Firestore listener never stopped; accumulates on view reopen
   - Impact: Firestore billing cost + memory leaks + 10x listeners firing after 10 view opens
   - Fix: Call stopObserving() in view onDisappear or auto-stop on auth logout

5. [MEDIUM] SSE Streaming String Concat O(n²) (F-perf-005)
   - Location: BereanChatView.swift:463
   - Problem: Every chunk appends via +=; for 2000 tokens = 2000 string ops
   - Impact: UI stutter on older devices; CPU spike during streaming
   - Fix: Buffer chunks locally; update UI every 5 chunks or 100ms in batches

6. [MEDIUM] Cross-Session History Requeried Every Send (F-perf-006)
   - Location: BereanChatView.swift:245-298
   - Problem: buildAllBereanHistory() fetches from Firestore fresh every send
   - Impact: Extra 200-500ms latency; compounds for users with 50+ conversations
   - Fix: Cache result for 5 minutes; invalidate on new message

7. [MEDIUM] WebSocket Audio Backpressure Throws (F-perf-007)
   - Location: BereanRealtimeWebSocketTransport.swift:51-65
   - Problem: appendInputAudio() throws immediately if buffer full; no backoff
   - Impact: Audio input cut off after 6-8 seconds on slow networks
   - Fix: Add async flow-control method that yields until buffer drains

LOW ISSUES (Nice-to-have optimizations)
---------------------------------------
8. [LOW] Model Router Evaluation Not Cached (F-perf-008)
   - Location: AmenAIModelRouter.swift:19-113
   - Problem: route() re-evaluates for same input+policy every time
   - Impact: Minimal (10-20ms per send); buried under preflight latency
   - Fix: Memoize decision per session; clear on new conversation
