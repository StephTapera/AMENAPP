Open format · Draft v0.1

.amen

A portable, signed context profile. Import the person, not the content.

Why this exists

Today, moving to a new platform means abandoning everything that made your old profile yours — or worse, dragging along years of posts, messages, and media you never wanted to relive. The .amen format does the opposite. It is a small, human-readable file that carries who you are: your interests, values, goals, skills, the communities you belong to, how you like to communicate, and where you are on your journey.

It deliberately carries none of your content. No messages, no posts, no media, no contact lists. That exclusion is the whole point — it is what lets .amen be a platform primitive that any app can read to welcome a person warmly without inheriting their old feed.

The facet shape

A .amen file is a list of facets. Each facet is one durable, self-described fact about the person. A facet never holds a sentence someone said or a thing they posted — only a stable trait, a list, or a preference. 

// one facet
{
  "category": "interests",        // one of 13 canonical categories
  "key":      "interest.ai",      // stable machine key (lowercase dotted)
  "label":    "Artificial intelligence",
  "value":    "Building thoughtful, human-first AI tools.",
  "visibility": "public"     // private | friends | groups | church | public
}

value is either a short free-text string (length-capped) or a list of short strings. Structured faith / communication / relationship facets are flattened to text or list when exported.

The 13 categories

interests, values, goals, skills, communities, relationships, communication, learning, faith_journey, current_focus, family, work, health.

Only your public self is exported

Every facet has a visibility. An export includes only facets you have marked public (or ones you explicitly check at export time). Sensitive facets — relationships, family, health, and the most private parts of your faith journey — live in the highest privacy tier and are never server-readable and never exportable.

The exclusion guarantee

This is enforced at two layers: the schema forbids any field that could hold content, and a unit-tested exclusion validator rejects any file whose values smuggle content shapes (emails, phone numbers, media URLs, contact arrays) into free text.

A .amen file MAY contain

  • Interests, values, goals, skills
  • Communities you belong to (by name)
  • Communication preferences
  • What you're learning / focused on
  • Public faith facets (favorite books, study)
  • A display name and public @handle

A .amen file MUST NOT contain

  • Messages or conversations
  • Posts or post text
  • Media files or media URLs
  • Email addresses or phone numbers
  • Contact lists or named contacts
  • Faith "areas needing support" Tier P

Schema-level structural enforcement: additionalProperties: false at every object (so no messages, media, or contacts array can ever be added), a 5-value visibility enum, a 13-value category enum, a handle pattern that forbids name@domain email shapes and + phone shapes, and length-capped values.

Signing

Every .amen file is signed by its issuer with Ed25519. The signature object names the algorithm, the key id (which resolves to a published public key), and the signature value. An importer verifies the signature before trusting the file, then routes every facet through the receiving app's Approval UI — nothing is silently written. 

{
  "alg":   "Ed25519",
  "keyId": "amen-export-2026-06",
  "value": "k3sJ8pQ2mRtVnW0xYz...=="   // base64 over the canonicalized file
}

Example file

A complete, signed .amen file with public facets only. This exact file validates against the v0.1 schema.

{
  "$schema": "https://amen.app/schemas/amen-context-v0.1.schema.json",
  "version": "0.1",
  "exportedAt": "2026-06-10T18:42:00Z",
  "issuer": "amen.app",
  "subject": { "displayName": "Jordan Rivera", "handle": "@jordanr" },
  "facets": [
    { "category": "interests", "key": "interest.ai",
      "label": "Artificial intelligence",
      "value": "Building thoughtful, human-first AI tools.",
      "visibility": "public" },
    { "category": "skills", "key": "skill.languages",
      "label": "Skills",
      "value": ["Swift", "TypeScript", "Product design"],
      "visibility": "public" },
    { "category": "faith_journey", "key": "faith.favorite_books",
      "label": "Books of the Bible I return to",
      "value": ["Psalms", "Philippians", "James"],
      "visibility": "public" }
  ],
  "signature": {
    "alg": "Ed25519",
    "keyId": "amen-export-2026-06",
    "value": "k3sJ8pQ2mRtVnW0xYz...=="
  }
}
