# Phase 2 — Agent G: BereanMessageTray + BereanCitationTile Fixes
2026-05-28

Agent G applied two ship-blocker accessibility fixes (A-15 and A-23 from the 03-a11y-perf-audit) plus two follow-on audit fixes (A-18, A-19). In `BereanMessageTray.swift`, an `.accessibilityActions { }` block was added directly to the outer `ZStack` container in `body`, exposing Copy, Regenerate, Share, Read aloud, and More actions as named VoiceOver custom actions — this resolves the ship-blocker that prevented VoiceOver users from accessing message tray actions at all, since `.onLongPressGesture` is inaccessible without pointer/touch. In `BereanCitationTile.swift`, the WCAG AA contrast failure (A-23, ~2.8:1) was resolved by replacing `Color.amenGold` with `BereanColor.textPrimary` on both the primary chip `Text(source.reference)` at 12pt and the cross-reference chip `Text(ref.reference)` at 11pt; `Color.amenGold` is retained exclusively for the ornamental verification badge icon and the confidence bar fill, where it carries no text-readability requirement. Both affected text views also gained `.dynamicTypeSize(.xSmall ... .accessibility3)` to support the full Dynamic Type range. As bonus fixes: the cross-reference `ScrollView` gained `.accessibilityLabel("Cross-references")` (A-18) and the `BereanCitationDetailSheet` Close button gained `.accessibilityLabel("Close citation").accessibilityHint("Dismisses citation detail")` (A-19); icon images inside cross-reference chips were marked `.accessibilityHidden(true)` to prevent VoiceOver from reading "book closed image" before the chip label. No new imports were added and all color tokens use `BereanColor.*` or the existing `Color.amenGold` ornamental usage.
