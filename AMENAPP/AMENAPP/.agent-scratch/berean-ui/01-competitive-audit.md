## Berean Strengths

1. **Personality mode system** — 13 distinct `BereanPersonalityMode` values (askBerean, scriptureStudy, prayerCompanion, discernment, scholar, coach, strategist, creator, builder, shepherd, deepStudy, mediaInsight, workLifeWisdom, safetyReview). Mode-switching is live, mid-chat, with animated aura response. No competitor matches this.

2. **Scripture intelligence layer** — Auto-detected citation chips, inline translation comparison (`AmenTranslationComparisonInline`), cross-reference suggestions, and verse preview sheet (`BereanVersePreviewSheet`) are domain-specific affordances with no parallel in Claude iOS or ChatGPT iOS.

3. **Provenance transparency** — Grok provenance chips show users exactly how each response was assembled (helper model, external search, sensitive detection). This exceeds Claude's and OpenAI's transparency at the message level.

4. **Study Mode reasoning visualization** — 9-node reasoning visualization (`BereanStudyModeSurface`, `BereanReasoningNode` with scripture / crossRefs / commentary / sermons / articles / originalLanguage / historicalContext / application / notes) is unique in the consumer AI chat market.

5. **Spiritual discernment pre-send intercept** — `BiblicalAlignmentService.shared.getDiscernmentPrompt` fires before each send, optionally routing the user through a framing choice. This is spiritually distinctive and has no analog in general-purpose AI apps.

6. **Memory scope control** — Four explicit scopes (off / thisChat / thisProject / allBerean) give users more intentional control over AI memory than either competitor.

7. **Church Notes integration** — Save to Notes, create note from response, and save outline to notes create a loop between AI conversation and the user's actual devotional record. Unique.

8. **Correct the AI (biblical lens)** — Inline per-response correction with optional memory save is a distinctive trust-building mechanic.

9. **Dynamic Island Live Activity** — Streaming indicator in the Dynamic Island during inference is polished and unique.

10. **Ghost draft restore** — Unsent drafts persisted via `BereanDraftStore` and surfaced as a dismissible chip. Small but thoughtful.

11. **Crisis safety short-circuit** — Pre-approved human-reviewed text (988 Lifeline, Crisis Text Line) bypasses AI sanitization entirely. More robust than competitors' generic safety banners.

12. **Ambient aura + wallpaper** — Mode-aware pastel aura blobs and user-selectable chat backgrounds add expressive depth absent from both competitors.

## Critical Gaps to Close

### Critical (blocks core utility)

**CG-1: No inline markdown rendering in the primary message path**
- Impact: Assistant responses with headers, bullet lists, numbered steps, and code arrive as unstyled plain text via `BereanStructuredResponseView`. For any structured study content this is a significant readability regression vs both competitors.
- Files: `BereanChatView.swift` line 2993, `BereanStructuredResponseView` — need `BereanMarkdownText` applied to the main `.content` path, not just the legacy `BereanChatBubble`.

**CG-2: Attachments not enabled ("not enabled for this build" alert)**
- Impact: The attachment flow exists (`BereanAttachmentPickerSheet`) and is wired to the composer, but the `showAttachmentsComingSoon` alert (line 1146–1150) blocks it entirely. Berean cannot process images or documents while both competitors can. Users who see the attachment button and get blocked will churn.
- Files: `BereanChatView.swift` line 1146; `BereanAttachmentPickerSheet.swift`

**CG-3: No ability to edit a sent user message**
- Impact: Both Claude iOS and ChatGPT iOS allow editing the last (or any) user message and re-running from that point. In Berean the only recovery path is the hidden context-menu Regenerate, which only re-runs the last turn, not the edited user text.
- Files: `BereanChatView.swift` — need an edit affordance on `userMessageBubble` (line 3079).

### High (materially degrades daily use)

**CG-4: Regenerate is hidden in context menu only**
- Both competitors surface Regenerate as a persistent visible button below the last assistant response. Berean's is buried 3 taps deep (long-press → context menu → "Regenerate"). This means most users will never find it.
- Files: `BereanChatView.swift` line 2547; add a persistent Regenerate icon to `BereanMessageTray` surface visible without long-press.

**CG-5: No last-message preview in conversations list**
- `BereanChatsListView` rows show title + translation badge + date but no message preview text. Both competitors show a truncated last message. Users cannot distinguish conversations at a glance.
- Files: `BereanChatsListView.swift` line 247–293; `BereanChatListItem` needs a `lastMessagePreview: String?` field.

**CG-6: No per-conversation delete (only Clear All)**
- The `clearAllRow` (line 344) destroys all conversations. There is no swipe-to-delete on individual `conversationRow` items. This is a data-safety regression — a user who wants to remove one conversation has no option short of destroying everything.
- Files: `BereanChatsListView.swift` line 234–293; add `.swipeActions` on each row.

**CG-7: Voice output (TTS read-aloud) is stubbed empty**
- `BereanMessageTray`'s `onAudio: {}` closure (line 3040) is an empty no-op. There is no audio playback of responses. ChatGPT iOS has read-aloud on every response. For scripture study, audio is a primary use case.
- Files: `BereanChatView.swift` line 3040; `BereanVoiceSpeechService.swift` exists and should be wired here.

**CG-8: Keyboard does not dismiss on scroll**
- No `.scrollDismissesKeyboard(.interactively)` modifier on the `ScrollView`. Standard in both competitors. When the keyboard is open and user scrolls up through messages, the keyboard blocks content.
- Files: `BereanChatView.swift` `contentScrollView` function, line 1577.

**CG-9: Alternative response selection (N/M arrows) absent**
- ChatGPT shows "2/3" arrows to cycle alternate responses for the same prompt without losing history. Berean's Regenerate destroys the existing response rather than appending an alternative. High cognitive cost for users iterating on a study question.

**CG-10: Response search in conversations list is title-only**

- Files: `BereanChatsListView.swift` line 31–34; needs full-text search via Firestore or local index.
